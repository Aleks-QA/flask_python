Основные команды GIT

Git — это система контроля версий, которая помогает отслеживать историю изменений в файлах.
  • git clone https://github.com/Aleks-QA/test.git Скопировать удаленный репозиторий
  • git status Проверка состояния репозитория
  • git add File1.txt Добавить файл в систему(Согласиться с изменением файла)
  • git commit "…" Коммит(«сохранить изменения») с указанием описания причин по которым это изменение было внесено
  • git push Отправить коммит на удаленный репозиторий
  • git fetch Загружает коммиты, файлы и ссылки из удаленного репозитория в локальный репозиторий (собирает все коммиты из целевой ветки, которых нет в текущей ветке, и сохраняет их в локальном репозитории. Однако он не сливает их в текущую ветку. )
  • git pull Команда pull сливает любые внесённые коммиты в ветку, в которой вы сейчас работаете, не давая вам сначала просмотреть их.
  • git log Посмотреть все коммиты(автор, дата, описание)
  • git log --author al694000n@gmail.com Посмотреть все коммиты автора(по имени или почте)
  • git blame File1.txt Какие изменения вносил тот или иной автор
  • git commit -a -m "…" «сохранить изменения» с указанием описания для всех файлов
  • git checkout first_branch Переключить на другую ветку+название той ветки
  • git log --pretty=oneline Посмотреть все коммиты, показывая каждый коммит в одну строку.
  • git revert 676ec Отменить коммит с идентификатором начинающимся на "676ec"
